üß† Project Overview

This web-based spam classifier uses a Machine Learning pipeline to classify user-submitted messages as either SPAM or HAM (not spam). It provides a confidence score based on the model's prediction probability.

üìÇ Dataset

  • Dataset: SMS Spam Collection Dataset
  • Source: UCI Machine Learning Repository
  • Samples: ~5,500 labeled SMS messages
  • Labels: 'ham' (legit) or 'spam'

‚öôÔ∏è Preprocessing Steps

  1. Convert to lowercase
  2. Tokenization using nltk.word_tokenize()
  3. Remove punctuation and special characters
  4. Remove stopwords using nltk.corpus.stopwords
  5. Stemming using PorterStemmer

üîç Feature Extraction

We use the TF-IDF Vectorizer to convert text data into numerical features that capture both word frequency and uniqueness.

ü§ñ Algorithm

We use the Bernoulli Naive Bayes classifier due to its efficiency and effectiveness in binary text classification.

  • Model Pipeline: TF-IDF Vectorizer ‚Üí BernoulliNB()
  • Training Accuracy: ~98%

üìä Evaluation Metrics

  • Precision: Proportion of predicted spam that is actual spam
  • Recall: Ability to detect all actual spam
  • F1 Score: Harmonic mean of precision and recall
  • Accuracy: Overall correct predictions
